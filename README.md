# syRun
to help application synchronized run on distributed system
# 说明
在单机中,加入同步锁的java代码可以按照顺序依次执行.
在分布式环境中,由于代码分布于不同主机,所以无法准确同步代码.
可以考虑将需要同步的代码移入同一服务器中顺序执行然后取结果.
或者通过同步锁的方式来执行代码.

1.分布式,多台服务器.
2.服务器里有执行接口,如http接口,传入序列化之后的任务信息,同步的key.最后返回数据.
3.可以根据key的hash来确定分发到哪一台具体的服务器.
4. 可以考虑用一个数据库保存具体的key导向的服务器的分配信息,这样几个保证多台服务器中的负载均衡.

redis的计数器和分布式事务能够一定程度上来实现此功能.

# 版本计划
## 0.X版本
此版本主要完成同步的基础功能,如下
- 同步原子计数器
- 同步原子锁实现
 - 普通同步请求,加入超时设置
 - 加入心跳包检测功能,实现掉线检查
- 通过请求,运行存放于服务器的指定代码功能
 - 通过实现指定的接口,来通过反射调用.
### 0.1版本

### 0.2 版本
